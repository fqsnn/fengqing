# 代码示例与项目写入边界

- “写一段代码”“示例代码”默认是回复内容，不是项目改动。
- Python 爱心示例走确定性快速路径，不依赖模型猜测，也不触发文件写入。
- 只有明确说明项目、文件或写入目标的改动请求，才进入智能体执行链。
- “继续推进自己的项目”是明确执行请求，优先进入受控自编程，而不是只解释下一步。
- “可以改一下自己的 UI 吗”是能力问句，必须说明桌面 UI 的独立范围，不得伪装成已完成修改。
- 未识别的智能体请求必须要求澄清目标，禁止默认回退为项目分析。
- 电脑操作必须诚实声明现状；没有可见确认、白名单、日志和停止机制时，不得声称可自动控制电脑。
- 联网搜索只发送用户明确提出的搜索词，不发送私人 workspace、私人记忆或完整对话内容。
- 项目写入仍必须经过用户授权、验证、日志和失败处理。
