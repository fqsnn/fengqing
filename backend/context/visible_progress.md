# 可见推进

用户反馈：感觉不到 AI 在推进什么。

系统修正：
- 推进不能只发生在代码和 GitHub 里，必须能被用户看见。
- 每次推进都应留下：当前阶段、完成证据、下一步、雨城映射。
- 前端应能显示进度，后端应提供进度接口。
- 用户问“你到底推进了什么”时，AI 应直接列出可验证变化，而不是泛泛解释。

当前阶段：
- AI 核心第三轮：行动。
- 普通对话可以受控调用智能体；私人记忆可以管理；两者共享可查询的本机活动历史。

本轮新增要求：
- 每次智能体执行结果必须附带 visible_progress。
- visible_progress 至少包含本轮步数、通过状态、写入状态、步骤证据和下一步。
- 写入权限、执行证据和历史必须持续可见，不能因为 AI 学会自行调用工具而隐藏。
- 下一步把确定性意图路由升级为结构化工具选择与任务状态机，并使用真实任务评测。
