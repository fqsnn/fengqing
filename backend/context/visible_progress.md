# 可见推进

用户反馈：感觉不到 AI 在推进什么。

系统修正：
- 推进不能只发生在代码和 GitHub 里，必须能被用户看见。
- 每次推进都应留下：当前阶段、完成证据、下一步、雨城映射。
- 前端应能显示进度，后端应提供进度接口。
- 用户问“你到底推进了什么”时，AI 应直接列出可验证变化，而不是泛泛解释。

当前阶段：
- AI 核心第二轮：记忆。
- 南京公开语境与本机私人记忆已经分层；私人事实可以从原话准确召回，默认不发送给远程模型。

本轮新增要求：
- 每次智能体执行结果必须附带 visible_progress。
- visible_progress 至少包含本轮步数、通过状态、写入状态、步骤证据和下一步。
- 下一步让记忆变化与智能体执行都拥有可查询、可修正、可删除的历史。
