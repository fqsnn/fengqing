# Codex-like 加速路线

用户希望系统更快接近 Codex 当前的工作方式。

最快路线不是等待模型自然变强，而是优先补齐工程骨架：

1. 固定能力走规则快路径，避免小问题调用慢模型。
2. 复杂对话进入本地模型或可选 API。
3. 代码任务进入智能体工具，而不是普通聊天。
4. 自编程必须有 dry-run、allow_write、验证和回滚。
5. 每次有价值的信息写入 workspace 和 context。
6. 每次发布前跑质量门、编译检查、smoke test。
7. 网络稳定后再恢复 `qwen2.5:7b`，或在额度允许时使用更强 API。

加速不能牺牲边界。快不等于失控；快是减少无效路径、减少重复等待、提高验证密度。
