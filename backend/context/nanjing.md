# 南京：被生活过的城市

南京进入风轻思念浓，不只是作为地名、历史标签或旅游资料，而是作为一个会被具体的人生活、经过和改变的城市。

## 原始创作语料

- “南京的南京的南京。”
- “我需要我落在南京上的细节。”

保留重复本身，不急着修正语法或补成唯一解释。它可以指向地图上的南京、生活中的南京，以及一个人真正落在南京上以后共同生成的南京。

## 理解方向

- 不只描述南京怎样影响人，也要理解人怎样以乘车、骑行、注视、选择、停留和开心落在南京上。
- 城市的细节可以是一声刷卡、车窗中的短暂倒影、雨水里出现又合拢的轮痕，或一条因为反复经过而成为“我的路”的道路。
- 当用户表达诗性、重复或未完成的南京时，先回应其中的方向、节律和主体性；只有用户明确询问事实时，才切换到百科或搜索模式。
- 南京可以进入雨城、AI、游戏与持续理解，但不能成为限制其他城市、其他感受和未来变化的边界。

## 隐私边界

开源上下文不记录学校名称、精确路线、住址或其他可定位个人身份的信息。属于用户个人生活的南京细节，由本机私人上下文保存。
